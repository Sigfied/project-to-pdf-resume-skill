# PDF 生产

本文件对应 `pdf-production.md`，用于生成或更新最终简历 PDF。

## 交付契约

只有具备以下内容时，PDF 任务才算完成：

- 源文件路径
- 渲染后的 PDF 路径
- 使用的编译/渲染命令或工具
- 页数
- 视觉检查结果
- 仍未确认的事实，如有

## 源文件选择

最终简历排版默认优先使用 LaTeX，除非有明确理由不用。LaTeX 适合作为默认方案，因为简历通常需要精确字体、稳定分页、紧凑间距、可重复构建和高质量 PDF 输出。

按优先级选择源文件：

1. 已经生成过当前简历的 LaTeX 源文件。
2. 只有当现有非 LaTeX 源文件明确是用户标准简历流程，或用户要求该格式时，才优先使用非 LaTeX。
3. 现有 PDF 以及旁边的源文件；可行时优先转成或维护为 LaTeX。
4. 将已有 Markdown 简历材料转换为 LaTeX。
5. 新建紧凑的 LaTeX 简历源文件。

除非用户要求重设计，否则保留当前视觉风格。

## 排版选择

这些情况默认使用 LaTeX：

- 用户需要精致 PDF
- 简历内容密集，需要精确控制间距
- 多语言字体处理重要
- 需要可重复构建
- 页数和换行需要精确控制

这些情况使用其他格式：

- 用户明确要求 DOCX、Markdown、HTML 或其他格式
- 现有非 LaTeX 源文件就是权威简历源
- 目标平台要求特定可编辑格式
- 本地没有 LaTeX 工具链且无法安装

使用 LaTeX 时，优先选择紧凑简历模板，使用语义化 section、可复用命令、一致的列表间距，并显式支持简历语言字体。

当用户提供现有简历 PDF 或 TEX 作为排版参考时，先检查这些样例再改版式。只提炼可复用排版规则：页面尺寸、边距、字号、section 层级、列表间距、项目标题结构、分页策略和视觉密度。不要把样例中的私人内容写进可复用 Skill 文档。

如果需要紧凑中文技术简历风格，读取 [layout-reference.cn.md](layout-reference.cn.md)。

## 源文件规则

- 使用支持简历语言的 LaTeX 引擎和模板；CJK 简历在可用时优先使用 XeLaTeX/LuaLaTeX 兼容字体处理。
- section 标题保持紧凑。
- 项目标题短一些：角色、组织/项目、时间即可。
- 先减少视觉杂乱，再考虑过度缩小字号。
- 没有视觉检查前，不要依赖脆弱的手动间距。
- 源文件和输出路径保持可预期。

## 排版前内容处理

动布局前先做：

1. 从最终 bullet 中删除未确认结论。
2. 按目标岗位相关性排序项目。
3. 缩短重复技术栈描述。
4. 把详细证据移到笔记，不塞进最终简历。
5. 确认名称、指标和敏感细节是否允许写出。

## 编译或渲染

优先使用项目已有的编译器或渲染器。LaTeX 场景下，优先使用仓库已有命令；没有时使用本地可用的可靠命令，例如 `tectonic` 或 `xelatex`。

编译或渲染失败时：

1. 读第一个真正错误，不只看最后摘要。
2. 修复缺包、不支持字符、字体配置、非法标记或布局溢出。
3. 重新编译或渲染。

LaTeX 文本中常见需要转义的字符：

| 字符 | 转义 |
| --- | --- |
| `%` | `\%` |
| `&` | `\&` |
| `_` | `\_` |
| `#` | `\#` |

非 LaTeX 格式也要遵守对应渲染器的转义和字体规则。

## 校验

声称完成前，必须检查实际渲染出的 PDF。

最低检查：

1. 确认 PDF 存在。
2. 确认页数符合预期。
3. 条件允许时，把页面渲染成图片。
4. 检查：
   - 字体渲染
   - 文本截断
   - 间距过紧或不均
   - section 平衡
   - bullet 是否越过边距
   - 孤行和别扭分页
   - 最强信息是否足够靠前

## 视觉 QA 清单

每页都检查：

| 区域 | 检查点 |
| --- | --- |
| Header | 姓名、角色、联系方式和链接清晰，不拥挤 |
| Summary | 目标岗位信号靠前 |
| Projects | 最强项目没有被埋没 |
| Bullets | 没有越界；缩进一致 |
| Sections | 标题有区分度但紧凑 |
| Typography | 字体对简历语言渲染正确 |
| Density | 页面充实但不拥挤 |
| Privacy | 敏感名称或数字按需泛化 |

常用命令因环境而异：

```bash
pdfinfo path/to/resume.pdf
pdftoppm -png -r 180 path/to/resume.pdf path/to/output-prefix
```

如果首选 PDF 渲染器不可用，用可用工具确认页数，并尽量用任何可用渲染器做视觉检查。

## 常见修复

| 问题 | 修复 |
| --- | --- |
| 多出一页 | 删除弱 bullet、重复技术栈和背景描述 |
| 项目块太密 | 拆长 bullet、压缩方法短语，或把细节移到笔记 |
| 首页弱 | 重排项目，或为目标岗位重写 summary |
| 行溢出 | 先缩短措辞，再考虑缩字号 |
| 字体渲染差 | 更换字体或编译/渲染设置 |
| 间距不一致 | 全局调整列表或 section 间距 |
| 影响不清晰 | 用已确认成果重写，或删除 |

## 迭代规则

- 如果 PDF 多出不想要的一页，先压缩弱内容和重复表述。
- 如果 bullet 太密，合并相关技术到一个短语。
- 如果第一页目标岗位信号不足，重排项目或重写 summary。
- 如果某行溢出，先缩短文案，再考虑缩字号。
- 如果换机器后渲染不同，检查字体和编译器设置。

只有内容、版式和校验都通过时才停止。编译成功本身不等于完成。

## 交付

返回：

- 最终 PDF 路径
- 源文件路径
- 修改摘要
- 仍需用户确认的事实

没有渲染并检查过，不要声称 PDF 已完成。
